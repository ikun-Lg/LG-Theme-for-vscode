# 🧈 小黄油 Little Butter — 设计系统

本文记录 **Little Butter（小黄油）** VS Code 主题的设计理念、配色体系与取舍依据，供后续迭代时保持一致。

---

## 1. 设计理念

> 一句话：**奶油黄做底，可可棕落墨，糖果色点睛，腮红粉互动。** 软萌不刺眼，可爱也要看得清。

四条核心原则：

1. **暖纸暖墨，绝不纯黑纯白。** 背景是柔和奶油黄 `#FCF4E2`，正文是温暖可可棕 `#5A4636`，而非 `#FFFFFF` / `#000000`。从明暗两端同时降低眩光，是长时间编码不累眼的关键。
2. **用色相区分语义，而非字重。** 每类语法分配一个独立、可辨识的「糖果」色，眼睛扫的是颜色不是粗体噪声。粗体几乎不用，只有 Markdown 标题/加粗保留。
3. **可爱与可读的平衡 —— "深糖果" 策略。** 纯粉彩在近白底上对比度只有 3:1 左右，会糊。所以采用 Catppuccin Latte 式的「加深的糖果色」（中低明度、中高饱和），保持萌感的同时让活跃语法色对比度 ≥ 4.0:1。
4. **配色取自主题图标。** 图标的三组核心色贯穿整个主题，图标与界面是同一套视觉语言。

---

## 2. 图标 → 主题的三组核心色

主题图标是一块带笑脸的奶油黄油，置于粉色底上。它定义了全局调色：

| 色组 | 取色 | 在主题里的角色 |
| --- | --- | --- |
| 🧈 **奶油黄油** | `#FCEDB6` `#F4CF6B` `#EFC152` | 当前行微光、活动栏、括号匹配、滚动条、函数（蜂蜜金）、链接 |
| 🌸 **腮红粉** | `#FBE0EA` `#F2A0B5` `#EC6CA4` | 光标、选区/查找、焦点框、标题栏、状态栏、悬停、按钮/徽章（更深的草莓粉 `#CC3D7F`） |
| 🍫 **可可棕** | `#5A4636` `#5A3A2A` | 正文/变量、标题与激活强调文字、眼睛/笑脸 |

## 3. 语法配色体系

所有活跃语法色都在奶油底 `#FCF4E2` 上验证过对比度（见 §6）。

| 类别 | 颜色 | 色相 | 说明 |
| --- | --- | --- | --- |
| 注释 Comment | `#94807B` *斜体* | 暖灰 | 刻意压到低对比「地板」，需要时看得到、扫读时不打扰 |
| 字符串 String | `#23824F` | 薄荷绿 | 唯一的绿，和函数的金、关键字的粉清晰区分 |
| 关键字 Keyword | `#C93A72` | 草莓粉 | 控制流锚点，主打可爱的品红粉 |
| 存储/修饰 Storage | `#B05530` | 焦糖 | 暖红棕，和关键字粉、函数金都拉开 |
| 函数 Function | `#947014` | 蜂蜜金 | 呼应黄油主体（文字需够深才清晰） |
| 类型/类 Type | `#8344C2` | 葡萄紫 | 冷色，从一众暖色里跳出来，"这是类型"一眼可辨 |
| 数字 Number | `#B5551E` | 蜜桃橙 | 字面量，区别于字符串绿与常量紫 |
| 常量/枚举 Constant | `#A33E8A` | 莓紫 | 与数字同属"字面量区"但更偏紫，可区分 |
| 变量 Variable | `#5A4636` | 可可棕 | 用正文色，不抢戏，颜色留给有语义的角色 |
| 参数 Parameter | `#B25468` | 玫瑰粉 | 比变量稍有色，低调区别于普通变量 |
| 属性 Property | `#2A78B5` | 天空蓝 | 对象成员/键，冷色让结构性内容后退 |
| 装饰器 Decorator | `#8743A6` | 紫罗兰 | `@xxx` 归到"元信息"紫，和类型紫稍作区分 |
| 标签 Tag | `#C84B34` | 珊瑚红 | HTML/JSX 标签，暖红但比关键字更橙、与之区分 |
| 操作符 Operator | `#7C6D5F` | 暖灰 | 连接性符号，弱化 |
| 标点/括号 Punctuation | `#8C7D6D` | 暖灰 | 结构性符号后退，让标识符主导 |
| 正则 Regexp | `#1F7E5F` | 青绿 | 比字符串绿更深一档 |

---

## 4. 字重与字形策略

- **斜体**：仅注释、`this`/`self`、Markdown 斜体。
- **粗体**：仅 Markdown 标题与加粗。
- **其余一律常规字重** —— 区分靠颜色，不靠粗细，字形节奏更平稳。

**品牌主交互色 = 腮红粉**（光标/焦点/选区/按钮/徽章）。把粉作为唯一的交互高亮，既统一又可爱；黄油负责底色与"注意力"表面（当前行、活动栏）。

## 6. 可读性 / 无障碍

- 目标：**活跃语法色对比度 ≥ 4.0:1**（近白底上做可爱主题的现实折中；AA 正文标准 4.5:1，AA-Large 3:1）。
- 正文 8.1:1、激活强调 9.25:1，远超标准。
- 注释、标点、装饰性符号刻意置于 3.4–3.7:1 的低对比"地板"，用于自然弱化（非疏忽）。
- 文字落在彩色填充上时取更深的同色系（如按钮/徽章用更深的草莓粉 `#CC3D7F` 配白字 4.36:1）。
- **验证工具**：`python3 .vscode/check_contrast.py themes/lggbond-butter-color-theme.json` —— 逐项打印每个语法色对编辑器背景的 WCAG 对比度，并标出低于阈值项。改色后务必跑一遍。

## 7. 构建与演进

- 主题文件：`themes/lggbond-butter-color-theme.json`（启用 `semanticHighlighting`，含 430+ 项 UI 着色 + 80 组语法作用域 + 语义高亮）。
- 调色由 `.vscode/build_butter.py` 通过统一的「色值重映射」从早期暖金浅色主题推导而来（奶油↔暖金、糖果粉↔金、糖果语法↔暖色语法）；图标源文件见 `.vscode/butter_icon.svg`。
- **历史**：项目早期为暖金「Sunflower」浅/深双主题，后整体转向可爱「小黄油」风，并于 v1.4.0 起改为单主题。旧配色可在 git 历史 v1.3.x 找回。

---

## 8. 改色清单（迭代时记得同步）

1. 改 `themes/lggbond-butter-color-theme.json` 的对应色值。
2. 跑 `.vscode/check_contrast.py` 验证对比度。
3. 若动了图标色，记得让三组核心色（黄油 / 腮红粉 / 可可棕）保持一致。
4. 升 `package.json` 版本 + 写 `CHANGELOG.md`。
5. `vsce package` 重新打包。
